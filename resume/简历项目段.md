# 简历项目段

## FanPulse - 欧美/K-pop 音乐追星内容运营工作台 ｜ 独立产品设计 & Vibe Coding ｜ 作品集项目

- **S：**欧美/K-pop 追星用户在回归宣发、流媒打歌、演唱会和日常安利中有大量内容生产需求，但粉丝站和普通粉丝经常依赖临时灵感，缺少稳定选题、发布节奏、多平台适配和数据复盘。
- **T：**设计一款面向音乐粉丝、粉丝站和内容运营同学的 AI 内容运营工作台，支持输入艺人、运营场景、平台和目标后，生成内容策略、多平台文案、发布日历、粉丝任务和复盘建议。
- **A：**拆解追星内容运营链路，将“热点节点 - 选题策划 - 文案生成 - 发布日历 - 粉丝任务 - 数据复盘 - 下一轮迭代”产品化，覆盖回归宣发、流媒任务、演唱会和日常追星 4 类场景。
- **A：**设计多平台内容生成模块，针对小红书、抖音/TikTok、X/Twitter 等平台输出不同语气和结构的内容，并提供标题、话题、视觉建议和评论区互动问题。
- **A：**从产品运营角度设计指标体系，覆盖方案生成完成率、内容复制率、发布日历使用率、互动率、收藏率、评论率和下一轮生成率，帮助内容运营从“凭感觉发帖”转向“可复盘迭代”。
- **A：**使用 Vibe Coding 完成可运行 Demo：前端实现动态音乐视觉界面、内容策略看板、发布日历、粉丝任务和复盘模块；后端使用 Node.js 预留 OpenAI Responses API 接口，并提供本地生成器兜底。
- **R：**完成 PRD、产品运营方案、指标体系、迭代路线图和 MVP Demo；项目可展示产品运营、内容社区、年轻用户洞察和 AIGC 应用能力。

## 产品经理版表达

- 独立设计欧美/K-pop 音乐追星内容运营工具，完成用户场景拆解、核心功能规划、多平台内容生成流程、数据复盘指标和 MVP Demo 实现。

## 产品运营版表达

- 围绕音乐追星场景设计内容运营闭环，覆盖回归宣发、粉丝任务、平台文案、发布日历和数据复盘，体现内容选题、用户分层、活动节奏和增长复盘能力。

## 和 ScentSoul 的组合逻辑

- ScentSoul：品牌产品、香水数字 IP、购买后会员运营。

两个项目一起放简历，可以体现：
