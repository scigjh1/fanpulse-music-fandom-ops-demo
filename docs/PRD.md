# FanPulse PRD

## 产品定位

FanPulse 是一款面向欧美/K-pop 音乐追星场景的 AI 内容运营工作台，帮助粉丝站和内容运营同学完成选题、文案、发布日历、粉丝任务和数据复盘。

## 用户与场景

### 目标用户

- K-pop / 欧美流行音乐粉丝。
- 粉丝站运营者。
- 音乐内容博主。
- 社区/内容运营实习生。

### 典型场景

- 回归宣发：新歌、新专辑、MV 上线前后的内容节奏。
- 流媒任务：播放、收藏、评论、分享、榜单目标。
- 演唱会巡演：抢票、穿搭、应援、现场内容。
- 日常追星：入坑安利、舞台推荐、歌词解读、数据复盘。

## 核心问题

1. 追星内容经常靠临时灵感，缺少稳定选题和发布节奏。
2. 同一个内容要适配多个平台，语气和格式差异很大。
3. 粉丝任务容易喊口号，但普通粉丝不知道今天具体做什么。
4. 内容发布后缺少复盘，无法判断下一条怎么优化。

## 产品方案

```text
输入艺人/场景/平台/目标
-> AI 生成策略
-> 生成多平台内容
-> 生成发布日历
-> 生成粉丝任务
-> 记录数据复盘
-> 给出下一轮迭代建议
```

## MVP 功能

| 模块 | 功能 | 价值 |
| --- | --- | --- |
| 项目配置 | 艺人、粉丝组织、场景、平台、目标 | 明确运营任务 |
| 策略生成 | 洞察、目标用户、传播钩子、定位 | 帮运营先想清楚 |
| 内容生成 | 小红书、短视频、X/Twitter | 适配多平台表达 |
| 发布日历 | D-7 到 D+7 的运营动作 | 管理回归节奏 |
| 粉丝任务 | 收藏、评论、分享、记录数据 | 把热情变成动作 |
| 数据复盘 | 曝光、互动、收藏、评论 | 用数据指导迭代 |
| 风险提醒 | 版权、引战、平台规则 | 降低追星内容风险 |

## 成功指标

- 内容生成完成率
- 内容复制率
- 发布日历使用率
- 粉丝任务完成率
- 内容互动率
- 收藏率
- 评论率
- 下一轮复盘使用率

## 合规边界

- 不使用未授权艺人图片、音源、饭拍素材。
- 不引导刷量、控评、攻击其他艺人。
- 不做虚假数据承诺。
- 重点做内容运营和粉丝协作，不做违规榜单操控。
